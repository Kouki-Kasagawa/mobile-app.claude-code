## 概要

v1 の学習タイマー・キャラ育成・カレンダー履歴に加え、「**疲労を可視化してあえて休むことを肯定する**」コアバリューを実装した。
外部 API 連携なし（v4 に分離）。予定は手動タグ入力のみ、疲労推定ロジックと UI の完成度に集中した。

## アーキテクチャの判断

### FatigueContext を新設（TimeContext は無変更）

TimeContext は「タイマー・学習時間」の単一責任を持つ。疲労ドメインを混在させると双方の保守が難しくなるため、**FatigueContext を別立て**にした。

プロバイダーの入れ子は以下の通り（既存ツリーに自然に接続）:

```
SettingsProvider（_layout.tsx）
  └── TimeProvider（(tabs)/_layout.tsx）
        └── FatigueProvider（(tabs)/_layout.tsx ← 追加）
              └── 各タブ画面
```

FatigueContext は TimeContext の `history` を直接参照せず、`useFatigueContext` 経由でフック側（`use-fatigue.ts`）が `useTime()` と組み合わせて前日スコア比較を行う設計にした。

## 疲労スコア計算ロジック（`utils/fatigueCalc.ts`）

### 2 軸スコア

| 軸 | 対象 | 計算式 |
|---|---|---|
| 身体的疲労 | 運動・部活・動くバイト・お出かけ 等 | `タグ重み × 時間(h)` |
| 認知的負荷 | 試験・課題・資格勉強・タスク過多 等 | `タグ重み × 時間(h) + ベースライン` |

### ベースライン加算（v1 の 7:1 換算を流用）

設定された学校/仕事時間帯を「内部的な認知負荷」として自動加算する。画面には表示しない。

```
ベースライン認知負荷 = 在席時間(h) ÷ workHoursPerStudy
```

SettingsContext の `workHoursPerStudy`（デフォルト 7）を共有することで、v1 換算ロジックと統一感を持たせた。

### タグ重み

| 重み | 意味 | 例 |
|---|---|---|
| 5 | 大 | 部活・試験 |
| 2 | 中 | バイト・課題 |
| 1 | 小 | お出かけ・買い物 |

### 閾値判定の優先順位（仕様 §2-1 に対応）

`buildFatigueState()` で以下の優先順位を適用する:

1. **① スコア合計が閾値（デフォルト 7）以上** → `danger`
2. **② 前日スコアが閾値以上 かつ 今日スコア > 0** → `ok` を `caution` に引き上げ
3. **③ 勉強記録が少なくても責めない** → UI 側が `wasYesterdayHigh` を見て文言を調整

## UI 調整（2026-06-29）

初回リリース後のフィードバックをもとに以下を修正した。

### 変更内容

| 項目 | 変更前 | 変更後 |
|---|---|---|
| 疲労スコアの数値表示 | スコア合計を `3.2点 / 7点` と表示 | 非表示（バーの視覚表現のみ） |
| AxisBar の数値ラベル | バー右端に `0.5` 等の数値を表示 | 非表示 |
| エントリのサブテキスト | `90分 · 7.5点` | `90分`（点数を除去） |
| セクション名 | 「今日の疲労要因」 | 「今日の予定・やること」 |
| モーダルタイトル | 「疲労要因を追加」 | 「予定・やることを追加」 |
| 空状態メッセージ | 「まだ要因を登録していません」 | 「まだ予定・やることを登録していません」 |
| 通常時 CTA | 「＋ 今日の疲労要因を追加する」 | 「＋ 今日の予定・やることを追加する」 |
| 疲労時 サブ CTA | 「要因を追加する」 | 「予定を追加する」 |

### 変更理由

- **スコア非表示**: 数値が見えると「7点以下だから大丈夫」と頭で判断して休息を先送りにしやすい。バーの視覚情報だけにすることで、直感的な疲労感の把握を促す。また「点数を稼ぐ/管理する」意識が生まれにくくなり、アプリの「休みを肯定する」思想に合う。
- **「疲労要因」→「予定・やること」**: ユーザーが日々の出来事を登録する感覚に近い言葉に変更。「疲労要因」は分析的・医療的なニュアンスがあり、日常的な入力の障壁になりうるため。

### 変更ファイル

| パス | 変更内容 |
|---|---|
| `app/(tabs)/fatigue.tsx` | スコア数値・AxisBar 数値ラベル削除、「疲労要因」→「予定・やること」の全文置換 |

## BlurOverlay の実装方針

React Native では Web の `filter: blur()` が使えないため、以下を組み合わせた:

1. **コンテンツ本体**: `opacity: 0.45` で彩度を下げる
2. **グレーオーバーレイ**: `rgba(140,140,140,0.35)` の半透明 View を absolute で重ねる
3. **expo-blur の BlurView**: `intensity={55}` でぼかし適用、`tint` はライト/ダーク対応

ライト/ダーク両対応は `useColorScheme()` で tint を切り替えることで対応。

## コアバリューへの一貫性

> 「学習アプリで唯一『休め』と言うアプリ」

実装した UI・文言はこの方針に準拠している:
- ❌ 「今日もサボったね」「もっと頑張れ」等の煽り文言は一切なし
- ✅ 「休むことも、続けるための戦略だよ。」等のポジティブな休息肯定メッセージ
- ✅ 勉強記録が 0 でも警告しない（`wasYesterdayHigh` で文言を緩める）
- ✅ 「やっぱり勉強する」で休息モードを自由に解除可能（強制しない）
